## 문항 2 정답지 — TF-IDF 직접 구현 및 코사인 유사도 기반 문서 검색

### 데이터 타입

| 필드 | 타입 | 설명 |
|------|------|------|
| `vocab_size` | int | 전처리 후 고유 단어 수 |
| `tfidf_matrix_shape` | list[int, int] | TF-IDF 행렬의 (문서 수, 단어 수) |
| `search_results` | list[dict] | 쿼리별 상위 3개 문서 검색 결과 |
| `search_results[].query` | str | 원본 쿼리 텍스트 |
| `search_results[].top3[].doc_index` | int | 문서 인덱스 (0-based) |
| `search_results[].top3[].similarity` | float | 코사인 유사도 (소수점 6자리 반올림) |

### 정답 체크리스트

| # | 체크 항목 | 배점 | 검증 내용 |
|---|----------|------|----------|
| 1 | num_documents 일치 | 10pts | 빈 줄을 제외한 문서 수가 정확한지 확인 |
| 2 | vocab_size 일치 | 20pts | NFC 정규화 적용 후 고유 단어 수가 일치하는지 검증 |
| 3 | tfidf_matrix_shape 일치 | 5pts | (문서 수 x 단어 수) 행렬 크기가 정확한지 확인 |
| 4 | 쿼리1 검색 결과 | 13pts | 상위 3개 문서 인덱스 및 유사도 일치 |
| 5 | 쿼리2 검색 결과 | 13pts | 상위 3개 문서 인덱스 및 유사도 일치 |
| 6 | 쿼리3 검색 결과 | 13pts | 상위 3개 문서 인덱스 및 유사도 일치 |
| 7 | 쿼리4 검색 결과 | 13pts | 상위 3개 문서 인덱스 및 유사도 일치 |
| 8 | 쿼리5 검색 결과 | 13pts | 모든 토큰이 불용어인 경우 유사도 0.0 처리 확인 |

- Pass 기준: 8개 체크 항목 모두 충족 (passing_score: 0.95)
- AI 트랩:
  - NFC 정규화를 소문자 변환 전에 적용해야 합니다 (순서가 바뀌면 vocab_size가 달라짐).
  - 빈 줄은 문서로 포함하지 않습니다 (documents.txt에 빈 줄이 있을 수 있음).
  - IDF 공식은 반드시 Smooth IDF(`log((N+1)/(df+1)) + 1`)를 사용해야 합니다.
  - 쿼리5는 전처리 후 모든 토큰이 불용어로 제거되어 영벡터가 되므로 유사도가 0.0이어야 합니다.

### 학습 목표 매핑

| 구현 요구사항 | 학습 목표 |
|-------------|----------|
| preprocess (NFC + 소문자 + 정규표현식 + 불용어) | 텍스트 전처리 파이프라인, 유니코드 정규화 이해 |
| TF 계산 | 단어 빈도 기반 문서 표현 이해 |
| Smooth IDF 계산 | 문서 빈도 역수의 의미와 스무딩 기법 이해 |
| TF-IDF 행렬 생성 | 희소 행렬 기반 문서-단어 벡터 공간 모델 구축 |
| cosine_similarity | 벡터 간 유사도 측정 수학적 원리 (내적, 노름) |
| search | 정보 검색 시스템의 쿼리-문서 매칭 파이프라인 |
